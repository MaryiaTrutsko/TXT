 Необходимые поля в баг-репорте.
  Project -"название проекта", ID - идентификатор отчета о дефекте (например, WS_VELC_20080625_000001),
Created Date - "дата создания отчета", Summary - "краткое описание, заголовок отчета, Что? Где? При каких условиях?",
Status - "состояние инцидента", Incident Type - "тип инцидента: дефект, запрос на улучшение". Также необходимо указать
Description, то есть описание (например - Если администратор заходит на страницу приветствия, то логотип пропадает).
